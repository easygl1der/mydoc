# 🚀 GitHub Pages 部署指南

## 快速开始

### 准备工作

确保你有以下文件：
- ✅ `index.html` - 网站主页
- ✅ 4个PDF文件
- ✅ `README.md` - 说明文档
- ✅ `.gitignore` - Git忽略文件

## 📝 详细步骤

### 第一步：创建GitHub账户

如果还没有GitHub账户：
1. 访问 https://github.com
2. 点击右上角 `Sign up` 注册
3. 完成邮箱验证

### 第二步：创建新仓库

1. 登录GitHub后，点击右上角的 `+` 号
2. 选择 `New repository`
3. 填写以下信息：
   - **Repository name**: 例如 `research-papers` 或 `pdf-library`
   - **Description**: 可选，例如"我的研究文档库"
   - **Public**: 必须选择公开（Private不支持GitHub Pages）
   - **不要**勾选 "Initialize this repository with a README"
4. 点击 `Create repository`

### 第三步：上传文件

#### 选项1：使用Git命令行（推荐）

如果你的电脑上已安装Git：

```bash
# 1. 打开命令行，进入当前目录
cd /path/to/研究

# 2. 初始化Git仓库
git init

# 3. 添加所有文件
git add .

# 4. 创建第一次提交
git commit -m "Initial commit: 添加PDF文档库网站"

# 5. 连接到GitHub仓库（替换成你的仓库地址）
git remote add origin https://github.com/你的用户名/你的仓库名.git

# 6. 推送到GitHub
git branch -M main
git push -u origin main
```

#### 选项2：使用GitHub桌面客户端

1. 下载并安装 [GitHub Desktop](https://desktop.github.com/)
2. 打开GitHub Desktop，登录你的账户
3. File -> Add Local Repository -> 选择当前文件夹
4. 点击 `Publish repository`

#### 选项3：直接网页上传（最简单）

1. 进入你刚创建的GitHub仓库页面
2. 点击 `uploading an existing file`
3. 将以下文件拖拽到浏览器窗口：
   - `index.html`
   - 所有4个PDF文件
   - `README.md`
   - `.gitignore`
4. 在底部填写提交信息：`添加PDF文档库网站`
5. 点击 `Commit changes`

### 第四步：启用GitHub Pages

1. 在仓库页面，点击顶部的 `Settings` 标签
2. 在左侧菜单中，找到并点击 `Pages`
3. 在 **Build and deployment** 部分：
   - **Source**: 选择 `Deploy from a branch`
   - **Branch**: 选择 `main` 分支，文件夹选择 `/ (root)`
4. 点击 `Save` 保存
5. 页面会显示：`Your site is ready to be published at https://你的用户名.github.io/仓库名/`

### 第五步：等待部署

- 首次部署通常需要 **1-5分钟**
- 刷新Settings -> Pages页面，看到绿色提示表示部署成功
- 点击链接访问你的网站！

## 🎉 成功！

你的网站现在已经上线了！网址格式为：

```
https://你的用户名.github.io/仓库名/
```

例如：
- 用户名是 `zhangsan`
- 仓库名是 `research-papers`
- 网址就是：`https://zhangsan.github.io/research-papers/`

## 🔄 更新网站

以后如果要更新PDF或修改网站：

### 通过Git命令行：
```bash
git add .
git commit -m "更新说明"
git push
```

### 通过网页：
1. 进入仓库
2. 点击要修改的文件
3. 点击编辑按钮（铅笔图标）
4. 修改后点击 `Commit changes`

更新后，GitHub会自动重新部署，通常1-2分钟后生效。

## ❓ 常见问题

### Q: 网站显示404错误
**A**: 
- 确保仓库是Public（公开）
- 确保GitHub Pages已在Settings中启用
- 等待5-10分钟让GitHub完成部署

### Q: PDF无法打开
**A**: 
- 确保PDF文件已成功上传到仓库
- 检查`index.html`中的文件名是否与实际PDF文件名完全一致（包括大小写）
- 中文文件名可能在某些浏览器中有问题，建议使用英文文件名

### Q: 想要自定义域名
**A**: 
1. 购买一个域名（如从阿里云、腾讯云等）
2. 在仓库根目录创建名为`CNAME`的文件，内容为你的域名
3. 在域名提供商处添加DNS记录指向GitHub Pages

### Q: 如何添加更多PDF？
**A**: 
1. 上传新的PDF文件到仓库
2. 在`index.html`中复制一个PDF卡片的代码
3. 修改标题和文件名
4. 保存并推送更新
